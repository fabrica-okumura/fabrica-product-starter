# 利用側ルール

## トリガー
- 同期対象のコンポーネント/スタイルを編集したい相談。
- B専用変更の配置先相談。
- `sync-list.json` の対象パスに作業が入るとき。

## 手順
1. 対象パスがAからの同期対象かを確認する。
2. 同期対象なら、そのファイルへの長期的な直接カスタマイズは避ける。
3. 可能なら同期対象外にB専用カスタマイズポイントを提案する。
4. 直接編集が避けられない場合は、同期期待へのリスクを明示する。

## ガードレール
- 同期対象ファイルをB所有の安定編集先として扱わない。
- 回答ではA/B責務の境界を明確に書く。
